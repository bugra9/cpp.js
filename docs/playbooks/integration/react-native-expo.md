# Integration — React Native + Expo

> Persona 2 sub-playbook. The user's project is an Expo-managed React Native app. Detection: `expo` (or `@expo/cli`) + `react-native` in deps.

## Goal

Add cpp.js to an Expo RN app so:

- iOS + Android native C++ compiles and links via Expo's prebuild system.
- Metro picks up the cpp.js loader.
- Both managed-prebuild and bare workflows are supported.

## When to use

- `expo` or `@expo/cli` in deps **and** `react-native` in deps.
- `app.json` (or `app.config.{js,ts}`) at root.
- Bare RN (no Expo) → `docs/playbooks/integration/react-native-cli.md`.

## Workflow note: Expo Go vs prebuild

cpp.js needs **native** code to ship with the app, so it requires:

- **Expo Prebuild** (`expo prebuild`) — generates `ios/` and `android/` dirs you commit (or regenerate from CI), then native libs link as in bare RN.
- **EAS Build** (`eas build`) — Expo's cloud builder; works the same as prebuild.
- **Expo Go** is **not** supported — it ships a fixed native binary; can't link new C++.

If the user is on Expo Go, you must walk them through `expo prebuild` first.

## Files involved

| File | Role |
|------|------|
| `package.json` | + `cpp.js`, `@cpp.js/plugin-react-native`, `@cpp.js/plugin-react-native-ios-helper`, `@cpp.js/plugin-metro`, optional `@cpp.js/package-<name>` |
| `metro.config.js` | Wrap Expo's `getDefaultConfig` with `CppjsMetroPlugin(...)` |
| `cppjs.config.{js,mjs}` *(new at root)* | Project-level cpp.js config |
| `app.json` / `app.config.{js,ts}` | No direct cppjs change; ensure `prebuild` is configured |
| `ios/`, `android/` | Generated by `expo prebuild`; native link picks up cppjs plugin |
| `src/native/` *(if user wraps own C++)* | `.h` + `.cpp` source |

## Commands

```bash
pnpm add cpp.js @cpp.js/plugin-react-native @cpp.js/plugin-react-native-ios-helper @cpp.js/plugin-metro
pnpm add @cpp.js/package-<name>     # optional

# Generate / regenerate native projects from app config
pnpm expo prebuild      # idempotent; safe to re-run

# iOS pods
cd ios && pod install && cd ..

# Run
pnpm expo run:android
pnpm expo run:ios
```

For EAS:

```bash
pnpm eas build --platform ios
pnpm eas build --platform android
```

EAS picks up `expo prebuild` artifacts automatically; cpp.js plugins kick in via the regenerated `Podfile` / `build.gradle`.

## Reference config

Mirror `cppjs-samples/cppjs-sample-mobile-reactnative-expo/`.

`metro.config.js` (canonical for Expo):

```js
const { getDefaultConfig } = require('expo/metro-config');
const { mergeConfig } = require('metro-config');
const CppjsMetroPlugin = require('@cpp.js/plugin-metro');

const defaultConfig = getDefaultConfig(__dirname);

const config = {
    ...CppjsMetroPlugin(defaultConfig),
};

module.exports = mergeConfig(defaultConfig, config);
```

`cppjs.config.mjs`:

```js
import Matrix from '@cpp.js/sample-lib-prebuilt-matrix/cppjs.config.js';

export default {
    dependencies: [
        Matrix,
    ],
    paths: {
        config: import.meta.url,
    },
};
```

## Multithread → COOP/COEP

**Not applicable for native RN.** RN runs JS on Hermes/JSC; multithread (`runtime: 'mt'`) uses pthreads via JSI. No browser headers required.

If the user also targets `expo-router` web build (RN Web), then they're effectively in a browser context and standard COOP/COEP rules apply — see `docs/playbooks/integration/vite.md` for headers.

## Validation

- [ ] `pnpm install` succeeds.
- [ ] `pnpm expo prebuild` generates `ios/` and `android/` without errors.
- [ ] `cd ios && pod install` succeeds.
- [ ] `pnpm expo run:ios` and `pnpm expo run:android` build + launch.
- [ ] App calls into C++: `import { initCppJs } from '@cpp.js/plugin-react-native'; await initCppJs(); Module.fn(...)` returns expected result.
- [ ] EAS build (if used) produces a valid binary.

## Common pitfalls

- **Trying to use Expo Go.** Won't work — cpp.js needs native compilation. Run `expo prebuild` first.
- **Forgetting to re-run `expo prebuild`** after adding native deps. Stale `ios/` / `android/` won't have the plugin's hooks.
- **Mixing Expo's `expo/metro-config` with bare `@react-native/metro-config`.** Use `expo/metro-config` for Expo projects (see canonical above).
- **`package.json` has both `expo` and stale RN-cli scripts.** Clean up: `expo run:android` replaces `react-native run-android` in Expo workflow.
- **EAS managed credentials prompt blocks CI.** When automating EAS, use `EAS_NO_VCS=1` or pre-set credentials.
- **arm64e / x86_64 simulator.** `@cpp.js/package-*-ios` podspecs already exclude `x86_64` for simulators. Custom packages must include the exclusion (see `react-native-cli.md` "Common pitfalls").
- **TypeScript paths broken.** `cppjs.config.{js,mjs,cjs}` is JS; ensure `tsconfig.json` includes it or excludes via `allowJs: true` if you want type checking.
